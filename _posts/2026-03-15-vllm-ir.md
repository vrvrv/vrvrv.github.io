# vLLM and torch.compile
kernel을 작성해서 모델 실행을 가속화하는데..
매번 custom kernel을 작성하는건 굉장히 시간이 오래걸리고 어려운일
torch가 jit compile을 하면 optimized kernel을 생성한다.


```python
@torch.compile
def fn(x, y):
    a = torch.sin(x)
    b = torch.cos(x)
    return a + b  

fn(x, y)
```
이런식으로 작성하면 torch가 알아서 최적화된 커널로 변환해서 실행

그럼 torch.compile은 어떻게 작동하는가 


(1) Frontend: Graph Capture (2) Backend: Optimization and kernel generation
여기서 중요한게, frontend의 torchDynamo는 컴파일이 불가능한 operation을 만나게 되면 graph break를 만든다는 것이다.
그런데 여기서 torch.compile이가느하고 안되고는 뭐지?? piecewiase cuda graph랑 차이는?
torchDynamo 에서 생성하는 graph break는 데이터 값에 따라 실행경로가 바뀌는 경우 graph break를 생성
아니면 torch 프레임워크를 벗어난 연산 (numpy, scipy, c++ 등)을 만난경



그다음으로는 backend에서는 torchInductor가 graph를 받아서 graph pass와 lowering을 진행 --> c++, triton 등의 커널 생


vllm의 custom compiler pass는?

1. 전체 아키텍처
torch.compile(model)
    │
    ▼
┌─────────────────────────┐
│  TorchDynamo (Frontend)  │  ← Python bytecode → FX Graph 캡처
└────────────┬────────────┘
             │ torch.fx.Graph
             ▼
┌─────────────────────────┐
│  TorchInductor (Backend) │  ← 그래프 최적화 + 커널 생성
│                          │
│  ┌───────────────────┐   │
│  │ Post-Grad Passes  │   │  ← ★ 여기에 custom pass를 끼운다
│  │ (CustomGraphPass)  │   │
│  └───────────────────┘   │
└─────────────────────────┘
vLLM은 torch._inductor.custom_graph_pass.CustomGraphPass를 상속해서 Inductor의 post-grad 단계에 자체 pass를 주입한다. Inductor가 그래프 최적화를 끝낸 뒤, 커널 코드 생성 직전에 이 pass들이 실행된다.

3. Custom Pass 작성법 — 두 가지 방식
방식 A: Inductor Pattern Matcher (패턴 매칭 + 치환)
"그래프에서 이 패턴을 찾아서 저걸로 바꿔라"는 선언적 접근. 가장 깔끔한 방법이다.
```python
from torch._inductor.pattern_matcher import (
    PatternMatcherPass,
    fwd_only,
    register_replacement,
)
from torch._higher_order_ops.auto_functionalize import auto_functionalized
from vllm.compilation.passes.inductor_pass import enable_fake_mode
from vllm.compilation.passes.vllm_inductor_pass import VllmInductorPass, VllmPatternMatcherPass
class MySiluQuantFusionPass(VllmPatternMatcherPass):
    """SiLU + FP8 Quantize → 하나의 fused 커널로 치환"""
    @enable_fake_mode  # 패턴 등록 시 실제 텐서 대신 fake tensor 사용
    def __init__(self, config):
        super().__init__(config)
        self.patterns = PatternMatcherPass(pass_name="my_silu_quant_fusion")
        # ── Step 1: 매칭할 패턴 정의 (before) ──
        def pattern(input: torch.Tensor, scale: torch.Tensor) -> torch.Tensor:
            # SiLU+Mul 먼저
            at1 = auto_functionalized(
                torch.ops._C.silu_and_mul.default,
                result=torch.empty(input.shape[:-1] + (input.shape[-1]//2,),
                                   device=input.device, dtype=input.dtype),
                input=input,
            )
            silu_result = at1[1]
            # 이어서 FP8 양자화
            at2 = auto_functionalized(
                torch.ops._C.static_scaled_fp8_quant.default,
                result=torch.empty_like(silu_result, dtype=torch.float8_e4m3fn),
                input=silu_result,
                scale=scale,
            )
            return at2[1]
        # ── Step 2: 치환할 패턴 정의 (after) ──
        def replacement(input: torch.Tensor, scale: torch.Tensor) -> torch.Tensor:
            d = input.shape[-1] // 2
            result = torch.empty(
                input.shape[:-1] + (d,),
                device=input.device,
                dtype=torch.float8_e4m3fn
            )
            # 하나의 fused 커널로 대체
            at = auto_functionalized(
                torch.ops._C.silu_and_mul_quant.default,
                result=result, input=input, scale=scale,
            )
            return at[1]
        # ── Step 3: 입력 텐서 생성 후 등록 ──
        inputs = [
            torch.empty(5, 64, device="cuda", dtype=torch.bfloat16),  # input
            torch.empty(1, 1, device="cuda", dtype=torch.float32),    # scale
        ]
        pattern(*inputs)  # 패턴 트레이싱 (반드시 호출해야 함)
        register_replacement(pattern, replacement, inputs, fwd_only, self.patterns)
    @VllmInductorPass.time_and_log
    def __call__(self, graph: torch.fx.Graph) -> None:
        # Inductor가 알아서 그래프를 순회하며 패턴 매칭 + 치환
        self.matched_count = self.patterns.apply(graph)
    def uuid(self) -> str:
        return self.hash_source(self)
```

핵심 포인트:
- `@enable_fake_mode`: __init__에서 패턴 등록 시 실제 GPU 텐서 대신 fake tensor를 사용
- `pattern()`: 기존 그래프에서 찾을 연산 시퀀스를 정의
- `replacement()`: 치환될 연산을 정의
- `register_replacement()`: pattern→replacement 매핑을 Inductor에 등록
- `__call__`에서 `self.patterns.apply(graph)` 호출하면 자동으로 매칭+치환

방식 B: 직접 FX Graph 조작 (수동 노드 순회)
패턴 매칭으로 표현하기 어렵거나, 단순 치환이 아닌 복잡한 변환이 필요할 때.
```python
class NoOpEliminationPass(VllmInductorPass):
    """불필요한 reshape/slice 제거"""
    @VllmInductorPass.time_and_log
    def __call__(self, graph: torch.fx.Graph) -> None:
        count = 0
        for node in graph.nodes:
            # ── reshape 체인을 base 텐서로 단축 ──
            if is_func(node, torch.ops.aten.reshape.default):
                input_node = node.args[0]
                if is_func(input_node, torch.ops.aten.reshape.default):
                    # reshape(reshape(x, s1), s2) → reshape(x, s2)
                    node.update_arg(0, input_node.args[0])
                    if len(input_node.users) == 0:
                        graph.erase_node(input_node)
                        count += 1
            # ── 동일 shape reshape/slice 제거 ──
            if is_func(node, torch.ops.aten.reshape.default):
                input_node = node.args[0]
                if self.shapes_equal(input_node, node):
                    node.replace_all_uses_with(input_node)
                    graph.erase_node(node)
                    count += 1
```
FX Graph 조작 핵심 API:
API	역할
graph.nodes	모든 노드 순회 (이터레이터)
node.args, node.kwargs	노드의 입력 인자
node.meta["val"]	텐서 shape/dtype 메타데이터
node.users	이 노드를 소비하는 다운스트림 노드들
node.replace_all_uses_with(other)	모든 사용처를 다른 노드로 교체
node.update_arg(idx, new_val)	특정 인자 교체
graph.erase_node(node)	노드 삭제 (사용처가 0이어야 함)
graph.inserting_before(node)	새 노드 삽입 위치 지정
graph.call_function(fn, args, kwargs)	새 함수 호출 노드 생성

4. Pass 등록 및 실행 흐름
PostGradPassManager가 모든 pass를 관리한다:
class PostGradPassManager(CustomGraphPass):
    def configure(self, config: VllmConfig) -> None:
        # config의 플래그에 따라 pass를 조건부 등록
        if self.pass_config.eliminate_noops:
            self.passes += [NoOpEliminationPass(config)]
        if self.pass_config.fuse_act_quant:
            self.passes += [ActivationQuantFusionPass(config)]
        if self.pass_config.fuse_norm_quant:
            self.passes += [RMSNormQuantFusionPass(config)]
        # ...
    def __call__(self, graph: fx.Graph) -> None:
        for pass_ in self.passes:
            pass_(graph)                    # 각 pass 순차 실행
        self.post_cleanup(graph)            # 정리
        self.fix_functionalization(graph)   # 항상 마지막에 실행
이 PostGradPassManager는 vLLM의 커스텀 compile backend(VllmBackend)에서 Inductor config에 주입된다:
# vllm/compilation/backends.py 내부
# Inductor의 post_grad_custom_pass 설정으로 PostGradPassManager를 등록
inductor_config["post_grad_custom_post_pass"] = pass_manager

6. 핵심 요약
측면	내용
진입점	CustomGraphPass (PyTorch Inductor 내장 확장 포인트)
베이스 클래스	InductorPass → VllmInductorPass → VllmPatternMatcherPass
패턴 매칭 방식	register_replacement(pattern_fn, replacement_fn, inputs, fwd_only, pass)
수동 조작 방식	graph.nodes 순회 → replace_all_uses_with / erase_node / update_arg
UUID	소스 코드 해시 기반 — pass가 변경되면 자동 재컴파일
실행 시점	Inductor post-grad, 커널 코드 생성 직전
등록	PostGradPassManager.add() 또는 inductor_config["post_grad_custom_post_pass"]
캐시 연동	uuid() 반환값이 Inductor code cache 키에 포함됨
요약하면: torch.compile의 Inductor 백엔드가 제공하는 CustomGraphPass 훅에 __call__(self, graph: fx.Graph) 메서드를 구현한 클래스를 끼워넣는 것이 전부다. 그 안에서 FX 그래프를 직접 조작하든, Inductor의 PatternMatcherPass로 선언적 치환을 하든 자유다.
